# 吃喝玩乐随机决策小程序

## 一、项目背景

在使用小红书收藏大量吃喝玩乐内容后，常见问题：

- 收藏内容越来越多，但经常遗忘
- 分类混乱，查找不方便
- 想出门时不知道选什么
- 反复纠结，决策效率低
- 做过的项目无法清晰标记
- 想再去一次的地方容易被淹没

因此，需要一个专门用于整理与随机决策的工具，提高收藏利用率与生活决策效率。

---

## 二、项目目标

打造一个：

- 可分类管理收藏内容
- 可查看、新增、编辑、删除
- 支持随机选择功能
- 支持状态管理（未做 / 已做 / 可再选）
- 提升决策效率，减少选择焦虑

核心价值：

> 让“选择困难”变成“一键随机决定”

---

## 三、目标用户

- 经常使用小红书收藏吃喝玩乐内容的人
- 选择困难人群
- 喜欢规划生活但缺少整理工具的人
- 想提高生活行动力的人

---

## 四、核心功能需求

### 4.1 分类管理模块

支持对吃喝玩乐内容进行分类管理。

#### 功能列表

- 查看所有分类
- 新增分类
- 编辑分类名称
- 删除分类
- 分类排序（可选）

#### 示例分类

- 🍜 吃饭
- ☕ 咖啡
- 🎬 娱乐
- 🌄 户外
- 🛍 购物
- 🎡 周边游

---

### 4.2 内容管理模块

每个分类下可管理具体项目。

#### 基础字段设计

| 字段 | 说明 |
|------|------|
| name | 项目名称 |
| image | 图片（可选） |
| note | 备注描述 |
| address | 地址（可选） |
| sourceLink | 来源链接 |
| status | todo / done / repeat |
| createTime | 创建时间 |
| updateTime | 更新时间 |

#### 支持功能

- 查看列表
- 查看详情
- 新增内容
- 编辑内容
- 删除内容
- 按状态筛选
- 按时间排序

---

### 4.3 随机选择功能（核心功能）

每个分类下支持：

- 🎲 随机选择一个项目
- 支持筛选随机范围：
  - 仅未做
  - 已做
  - 可再选
  - 全部

#### 默认规则

- 默认优先从“未做”中随机
- 若未做为空，可提示从“可再选”中选择
- 支持“重新随机”

#### 体验优化建议

- 抽签动画
- 随机历史记录
- 决策结果强调展示

---

### 4.4 状态管理系统

每个项目支持以下状态：

| 状态 | 说明 |
|------|------|
| todo | 未做 |
| done | 已做 |
| repeat | 可再选 |

#### 状态流转逻辑

- 随机完成后可手动标记为 done
- done 可改为 repeat
- repeat 可改为 done

---

## 五、使用流程示例

1. 在小红书看到想去的内容 → 添加到小程序
2. 归类到对应分类
3. 想出门时：
   - 进入分类
   - 点击“随机”
   - 直接决定
4. 体验后 → 更新状态
